Getting Started with Linux Kernel Development

December 28, 2025

Kernel development has a reputation for being difficult to approach. The codebase is large, the tooling is different from userspace work, and mistakes can crash your machine. However, getting started is more straightforward than it appears once you have the right setup.

This article covers the initial steps: setting up a development environment, building the kernel, and writing a basic module.

Prerequisites

Before starting, you should be comfortable with C programming and have basic familiarity with Linux system administration. You will need a machine or virtual machine running a recent Linux distribution. Using a VM is recommended for initial work since kernel bugs can require a reboot.

Install the required packages on Debian-based systems:

sudo apt install build-essential libncurses-dev bison flex libssl-dev libelf-dev

On Fedora or RHEL-based systems:

sudo dnf install gcc make ncurses-devel bison flex elfutils-libelf-devel openssl-devel

Getting the Source

Clone the mainline kernel repository. This takes time and disk space, around 3-4 GB:

git clone https://git.kernel.org/pub/scm/linux/kernel/git/torvalds/linux.git
cd linux

For a smaller download, use a shallow clone with recent history only:

git clone --depth 1 https://git.kernel.org/pub/scm/linux/kernel/git/torvalds/linux.git

Configuration

The kernel has thousands of configuration options. For development purposes, start with your current running kernel configuration:

cp /boot/config-$(uname -r) .config
make olddefconfig

The olddefconfig target applies defaults for any new options not in your existing config.

Building

Compile the kernel using all available cores:

make -j$(nproc)

This takes 15-60 minutes depending on hardware. A successful build produces vmlinux and arch/x86/boot/bzImage (on x86 systems).

Writing a Simple Module

Kernel modules are the entry point for most new contributors. They load and unload without rebooting, making iteration faster.

Create a file named hello.c:

#include <linux/init.h>
#include <linux/module.h>
#include <linux/kernel.h>

MODULE_LICENSE("GPL");
MODULE_AUTHOR("Your Name");
MODULE_DESCRIPTION("A minimal kernel module");

static int __init hello_init(void)
{
    printk(KERN_INFO "Hello from the kernel\n");
    return 0;
}

static void __exit hello_exit(void)
{
    printk(KERN_INFO "Goodbye from the kernel\n");
}

module_init(hello_init);
module_exit(hello_exit);

Create a Makefile in the same directory:

obj-m += hello.o

all:
	make -C /lib/modules/$(shell uname -r)/build M=$(PWD) modules

clean:
	make -C /lib/modules/$(shell uname -r)/build M=$(PWD) clean

Build and test the module:

make
sudo insmod hello.ko
dmesg | tail
sudo rmmod hello

You should see your messages in the kernel log output.

Next Steps

With a working development setup, you can explore the kernel source, read documentation in the Documentation/ directory, and look for simple bugs to fix. The kernel newbies mailing list and staging drivers are good places to find tasks suitable for new contributors.

Start small, test thoroughly, and read the existing code before making changes.
